# DNS Shop Microservices

Учебный проект имитации магазина DNS на микросервисной архитектуре. Все компоненты собираются и запускаются внутри **Docker**.

## Структура проекта

*   **user-service** (порт 8080) — управление пользователями (Java 21, Spring Boot 3.4+, MapStruct, WebFlux, Liquibase, PostgreSQL 16, R2DBC).
    *   **Фичи**: Регистрация с профилем в одной транзакции, **Optimistic Locking** (@Version), интернационализация ошибок (i18n), частичное обновление (PATCH), мапперы с игнорированием `null`.
*   **product-service** (порт 8082) — каталог товаров и категорий.
    *   **Фичи**: Реактивный **JOIN** (Товар + Категория), гибкие характеристики через **JSONB**, динамические фильтры через `DatabaseClient`, пагинация, **аналитика фильтров** (подсчет количества товаров по атрибутам внутри JSONB).

## Эндпойнты системы

### User-Service


| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/v1/users` | Получить список всех пользователей по фильтру |
| `GET` | `/api/v1/users/{id}` | Получить полную информацию о пользователе (User + Profile) |
| `POST` | `/api/v1/users` | Регистрация нового пользователя |
| `PATCH`| `/api/v1/users/{id}` | Частичное обновление профиля (игнорирование null) |

### Product-Service


| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/v1/products` | Поиск товаров с пагинацией и фильтрами (цена, имя, JSON-атрибуты) |
| `GET` | `/api/v1/products/{id}` | Получить товар с названием категории и атрибутами |
| `GET` | `/api/v1/products/filters` | Получить дерево доступных фильтров с количеством товаров для категории |
| `POST` | `/api/v1/products` | Создать новый товар (с проверкой категории) |
| `PATCH`| `/api/v1/products/{id}` | Частичное обновление данных товара и JSON-атрибутов |
| `POST` | `/api/v1/categories` | Создать категорию товара |

## Тестирование

*   **Интеграционные**: Настроен **Singleton Testcontainers**, статический запуск для стабильности соединений, автозаполнение и очистка таблиц перед каждым тестом.
*   **Параметризованные**: При тестировании фильтрации и валидации.
*   **Jacoco**: Покрытие бизнес-логики > 80%.

## План развития (Backlog)

1.  **Улучшение поиска (Product-Service)**: Поддержка диапазонов (gt/lt) для числовых характеристик в JSONB.
2.  **Оптимизация фильтров**: Внедрение **Redis** или фоновых **Jobs** для кэширования счетчиков фильтров при больших объемах данных.
